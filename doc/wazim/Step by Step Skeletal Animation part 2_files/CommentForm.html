    Reply: Laura

    The COLLADA 1.4 specification does not clearly say what the time value is. But different COLLADA exporters interprets that as seconds. So 0.001 is one mili-second in time but its represented as seconds. This increases with constant values because your frames are always apart in constant time from each other. If you have an animation which is running 1000 frames a second, then the next frame will be on 0.002 time and so on.
    So basically each time value is the time for that frame in "real time".

    I just read the code and understood it once again. I guess thats why people say comments are a good thing :).

    First of all the this->m_AnimationTime variable is used in many places, specially in CCBA::CalculateCurrentFrame.

    The m_AnimationTime variable is used to keep record of where we are in the animation timeline in seconds. For example we have an animation timeline of 6 seconds and the animation is running for 2 seconds we should be at the 2 seconds in the animation timeline. If the animation now runs for more 3 seconds we should now be at 5 seconds. Thats all what m_AnimationTime is keeping track of and it wraps to zero after it passes the total time.

    frame = (int)(this->m_AnimationTime * (float)this->m_SkeletonData->m_NoOfKeyframes / this->m_Header->m_AnimationChunk.m_TotalTime)

    The above line finds out the keyframe just before the current m_AnimationTime. If you recall, we are given animation keyframes. But the m_AnimationTime is the current elapsed time which might not be on a keyframe. This is why its cast to an (int). If you want the actual current time elapsed you don't need the cast. This works because we are lucky and we know that we are given keyframes with same delta. Each next keyframe is on LastKeyframe + DeltaTime.

    Its calculated using Cross-Multiplication. Wikipedia Cross-Multiplication

    You can read the above line as follows.
    If in m_TotalTime there are m_NoOfKeyframes
    Then in m_AnimationTime how many frames are there?
    m_TotalTime-----------------m_NoOfKeyframes

    m_AnimationTime-----------------how-many-frames

    And then you cross multiply them.

    how-many-frames = ((m_AnimationTime * m_NoOfKeyframes) / m_TotalTime);

    After that if you read the SetupSkeleton method. You will find that we are also calculating inBetween

    inBetween = (float)(this->m_AnimationTime * (float)this->m_SkeletonData->m_NoOfKeyframes / this->m_Header->m_AnimationChunk.m_TotalTime)

    You don't need this (float) cast here. Here we are doing the same thing but now we need the actual fractional elapsed time, instead of the keyframe before the actual elapsed time. This inBetween is now the actual time in the current animation timeline. If you deduct the current keyframe from this value you will get the fraction of time passed over the current keyframe time. This fraction is then used to interpolate the matrices between the current and next keyframe. Please read about interpolation in the above comments.
